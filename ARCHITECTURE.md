# AI剧本协同创作系统 - 架构设计说明

## 项目概述

本项目是基于AI原生架构的专业剧本协同创作系统，旨在解决传统剧本创作中的效率和质量问题，通过AI技术赋能编剧创作过程。

## 技术栈

- **前端**：Vue3 + Element Plus + ECharts
- **后端**：Python FastAPI
- **数据层**：Neo4j（可选，用于叙事状态网络）
- **AI能力**：GPT-4o + 智谱清言
- **部署**：本地启动脚本 + 可选静态前端构建

## 核心创新点

### 1. 叙事规则内生驱动的AI剧本生成双内核架构

**技术实现**：
- **规则引擎层**：基于Neo4j图数据库构建叙事规则图谱，存储结构化的叙事规则
- **AI生成层**：封装GPT-4o和智谱清言的标准化调用接口，通过Function Call实现核心逻辑
- **双内核协同**：规则引擎提供结构化约束，AI生成层提供创意内容，两者通过中间层实现协同

**关键模块**：
- `backend/app/api/ai.py`：人物、剧情大纲、首幕生成接口
- `backend/app/api/narrative.py`：按幕续写、完成度判断、叙事状态同步
- `backend/app/core/rule_engine.py`：叙事规则校验
- `backend/app/core/story_constraints.py`：题设锚点抽取与对齐约束
- `backend/app/models/neo4j_db.py`：叙事状态网络数据构建

### 2. 双轨联动的人机共生创作范式

**技术实现**：
- **人工创作轨**：基于Monaco Editor的专业文本编辑组件，支持实时协作
- **AI辅助轨**：提供智能建议、自动补全、风格一致性检查等功能
- **联动机制**：通过WebSocket实现实时同步，AI根据人工输入动态调整建议

**关键模块**：
- `frontend/src/views/PipelineView.vue`：自动化工坊
- `frontend/src/views/EditorView.vue`：显性创作轨
- `backend/app/api/narrative.py`：幕生成、幕审查与修订API

### 3. 基于叙事指纹的剧本全链路价值保障体系

**技术实现**：
- **叙事指纹提取**：基于Milvus向量数据库存储剧本的语义向量
- **版权保护**：通过叙事结构特征生成唯一指纹，用于版权验证
- **价值评估**：建立层级化叙事结构评估体系，量化剧本质量

**关键模块**：
- `backend/app/api/fingerprint.py`：叙事指纹接口
- `frontend/src/views/FingerprintView.vue`：叙事指纹展示页

## 数据流程

1. **项目创建**：用户创建新项目，存储到MySQL
2. **规则配置**：系统加载叙事规则到Neo4j
3. **剧本创作**：
   - 人工编辑剧本内容
   - AI基于规则和上下文提供建议
   - 实时协作同步
4. **质量评估**：
   - 提取叙事指纹到Milvus
   - 评估剧本质量和一致性
5. **导出交付**：生成最终剧本文件

## 核心API接口

### 项目管理
- `POST /api/projects`：创建项目
- `GET /api/projects`：获取项目列表
- `GET /api/projects/{id}`：获取项目详情
- `PUT /api/projects/{id}`：更新项目
- `DELETE /api/projects/{id}`：删除项目

### 人物管理
- `POST /api/characters`：创建人物
- `GET /api/characters/project/{project_id}`：获取项目人物列表
- `GET /api/characters/{id}`：获取人物详情
- `PUT /api/characters/{id}`：更新人物
- `DELETE /api/characters/{id}`：删除人物

### 叙事规则
- `POST /api/narrative/rules`：创建叙事规则
- `GET /api/narrative/rules`：获取叙事规则列表
- `POST /api/narrative/apply-rule`：应用叙事规则

### AI服务
- `GET /api/ai/runtime-settings`：读取当前运行时模型配置
- `POST /api/ai/runtime-settings`：保存当前运行时模型配置
- `POST /api/ai/narrative/characters`：生成人物设定
- `POST /api/ai/narrative/outline`：生成题材对应大纲
- `POST /api/ai/narrative/script`：生成第一幕正文

## 启动步骤

1. **环境准备**：
   - 安装Docker和Docker Compose
   - 配置API密钥（OpenAI和智谱AI）

2. **构建和启动**：
   ```bash
   cd ai-screenplay-system
   docker-compose up --build
   ```

3. **访问系统**：
   - 前端：http://localhost
   - 后端API文档：http://localhost:8000/docs

4. **初始化数据**：
   - 访问后端API文档，调用初始化接口创建默认叙事规则
